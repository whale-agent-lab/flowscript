# Artifact Contract

The run directory is `runs/{run_id}/`. Generated data and analysis artifacts are stored separately: `data/` contains script-generated data consumed by later steps, while `artifacts/` contains profiles, interpretations, status files, and the final report.

Key files:

- `valid_params.json`: Normalized input containing employees, regions, the grouping dimension, and an explicit inclusive date range.
- `data/generated_data.csv`: Synthetic historical data generated by the script, with order, employee, region, amount, status, and date columns. It is neither user input nor the final deliverable.
- `artifacts/profile.json`: Date-filter results, field profiles, missing values, unique values, numeric statistics, duplicate rows, and grouping statistics.
- `artifacts/interpretation.json`: English summary, risks, recommendations, and grouping insights.
- `artifacts/final_report.md`: Primary deliverable.
- `artifacts/*_status.json`: Structured status files.
- `artifacts/fallback_context.json`: Failed step, completed steps, available artifacts, error logs, and English recovery actions.
- `artifacts/artifacts_manifest.json`: Final artifact manifest.

Scripts write JSON as UTF-8. Machine states and all user-facing messages are in English. Never describe synthetic data as real employee-performance data.
